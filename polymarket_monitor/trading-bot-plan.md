# 자동매매봇 플랫폼 개발 계획

## 📋 프로젝트 개요

**프로젝트명:** CryptoBot Studio (개미들의 자동매매 봇 빌더)  
**타겟:** 암호화폐 소매 투자자("개미")  
**목표:** 개인 투자자가 직접 거래 전략을 설정하고 백테스팅한 후 실거래할 수 있는 플랫폼  
**핵심 가치:** 저가격 + 한국어 + 초보자 친화적 + 투명성

---

## 🎯 핵심 기능 (MVP)

### Phase 1: 기본 백테스팅 엔진 (1단계)

#### 1.1 전략 설정 UI

**버튼 1: RSI 기반 매도 전략**
- 이름: "RSI 과매도 매도"
- 설명: RSI가 90 이상일 때 자동 매도
- 파라미터:
  - RSI 기간: 14 (기본값, 수정 가능)
  - 상한선: 90 (기본값, 수정 가능 80-100)
  - 거래량: 포지션의 100% (수정 가능)
  - 수익률 목표: 사용자 입력값
- 로직: 
  ```
  IF RSI > 상한선 THEN 매도
  ```

**버튼 2: RSI 기반 매수 전략**
- 이름: "RSI 과매수 매수"
- 설명: RSI가 30 이하일 때 자동 매수
- 파라미터:
  - RSI 기간: 14 (기본값, 수정 가능)
  - 하한선: 30 (기본값, 수정 가능 0-30)
  - 거래량: 초기 자본의 X% (사용자 입력)
  - 손절 설정: Y% (사용자 입력)
- 로직:
  ```
  IF RSI < 하한선 THEN 매수
  ```

**버튼 3: 볼린저밴드 하단 매수 전략**
- 이름: "BB 하단 터치 매수"
- 설명: Bollinger Bands 하단에 닿으면 자동 매수
- 파라미터:
  - BB 기간: 20 (기본값, 수정 가능)
  - 표준편차: 2 (기본값, 수정 가능 1-3)
  - 거래량: 초기 자본의 X% (사용자 입력)
  - 손절: Y% (사용자 입력)
- 로직:
  ```
  IF Close Price <= BB_Lower THEN 매수
  ```

**버튼 4: 볼린저밴드 상단 매도 전략**
- 이름: "BB 상단 터치 매도"
- 설명: Bollinger Bands 상단에 닿으면 자동 매도
- 파라미터:
  - BB 기간: 20 (기본값, 수정 가능)
  - 표준편차: 2 (기본값, 수정 가능 1-3)
  - 수익 실현: Y% (사용자 입력)
- 로직:
  ```
  IF Close Price >= BB_Upper THEN 매도
  ```

#### 1.2 API 키 입력

**API 설정 Input 박스:**
- 거래소 선택: Dropdown (Upbit, Bithumb, Binance)
- API Key Input: "*****" 마스킹 표시
- Secret Key Input: "*****" 마스킹 표시
- 테스트 연결 버튼: "API 연결 테스트"
  - 성공: ✅ 계좌 잔액 표시
  - 실패: ❌ 오류 메시지
- 저장 버튼: 로컬 암호화 저장 (localStorage)
- 보안 공지: "API 키는 절대 공개되지 않으며, 로컬에만 저장됩니다"

#### 1.3 백테스트 실행

**버튼 5: 백테스트 실행**
- 클릭 시 프로세스:
  1. 히스토리컬 데이터 로드 (기본 1년, 사용자 수정 가능)
  2. 선택된 전략 적용
  3. 거래 시뮬레이션
  4. 결과 계산
- 로딩 상태: 스피너 + "백테스팅 중... (진행률)"
- 실행 시간: 1-3초

## 📊 데이터베이스 스키마

### Users 테이블
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY,
  nickname VARCHAR(50) UNIQUE NOT NULL,
  email VARCHAR(100) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  profile_image_url VARCHAR(255),
  bio TEXT,
  level INT DEFAULT 1,
  badges JSON, -- ["검증됨", "핫", "전설"]
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  updated_at TIMESTAMP,
  deleted_at TIMESTAMP -- soft delete
);
```

### Strategies 테이블
```sql
CREATE TABLE strategies (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id),
  title VARCHAR(100) NOT NULL,
  description TEXT,
  strategy_type VARCHAR(50), -- "RSI_SHORT", "RSI_LONG", "BB_BUY", "BB_SELL", "CUSTOM"
  parameters JSON, -- {rsi_period: 14, rsi_threshold: 30, ...}
  backtest_result JSON, -- {profit_rate: 45.32, win_rate: 65.2, ...}
  tags JSON, -- ["RSI", "초보자", "트렌드"]
  likes_count INT DEFAULT 0,
  comments_count INT DEFAULT 0,
  views_count INT DEFAULT 0,
  is_public BOOLEAN DEFAULT true,
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  deleted_at TIMESTAMP
);
```

### Comments 테이블
```sql
CREATE TABLE comments (
  id UUID PRIMARY KEY,
  strategy_id UUID NOT NULL REFERENCES strategies(id),
  user_id UUID NOT NULL REFERENCES users(id),
  parent_comment_id UUID REFERENCES comments(id), -- 답글용
  content TEXT NOT NULL,
  rating INT CHECK (rating >= 1 AND rating <= 5), -- 1-5 별점
  likes_count INT DEFAULT 0,
  is_deleted BOOLEAN DEFAULT false,
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  deleted_at TIMESTAMP
);
```

### Backtests 테이블
```sql
CREATE TABLE backtests (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id),
  strategy_id UUID REFERENCES strategies(id),
  parameters JSON NOT NULL,
  result JSON NOT NULL, -- {profit_rate, win_rate, mdd, sharpe_ratio, ...}
  execution_time_ms INT,
  created_at TIMESTAMP
);
```

### Likes 테이블
```sql
CREATE TABLE likes (
  id UUID PRIMARY KEY,
  user_id UUID NOT NULL REFERENCES users(id),
  strategy_id UUID NOT NULL REFERENCES strategies(id),
  created_at TIMESTAMP,
  UNIQUE(user_id, strategy_id)
);
```
